# Security Policy

## 支持范围

当前项目处于早期阶段，默认只维护 `main` 分支上的最新版本。

## 报告安全问题

如果你发现安全问题，请不要在公开 Issue 中贴出真实 API Key、私有中转站地址或完整敏感响应。

建议通过 GitHub 私有安全通道报告。如果仓库尚未启用私有安全报告，请先创建一个不含敏感细节的 Issue，说明你希望私下报告安全问题。

## 密钥处理

- API Key 不会写入磁盘。
- API Key 会从浏览器发送到本地服务，再转发到用户填写的 `Base URL`。
- 浏览器页面不会主动连接除本地服务以外的地址。
- 使用者需要自行确认 `Base URL` 属于可信服务。

## 发布和反馈注意事项

- 不要提交包含真实 API Key 的截图、日志或测试文件。
- 不要把私有中转站地址写入公开文档。
- 如果误提交了密钥，请立即吊销并重新生成。
